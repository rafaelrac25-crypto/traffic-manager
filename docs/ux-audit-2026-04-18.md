## 🔴 Crítico

### 1. Sino → data comercial não leva ao anúncio pré-preenchido
**Onde:** `frontend/src/contexts/AppStateContext.jsx:138`
**O que acontece:** Quando o sistema gera uma notificação "data comercial chegando", o link da notificação é `/calendario`. A Cris clica no sino → vai pro calendário → tem que encontrar o dia → abrir modal → clicar "Criar anúncio". **4 cliques do sino até o CreateAd.**

**Por que dói na Cris:** Ela abriu o app entre atendimentos justamente pelo alerta. Cada clique extra é fricção na janela de atenção dela.

**Conserto (1 linha):**
```diff
- link: '/calendario',
+ link: '/criar-anuncio',
+ state: { commercialDate: { ... } },
```
O payload `commercialDate` com `dateISO`, `daysBefore`, `preFill` já existe no `commercialDates.js` — é só passar no state da notificação e o `CreateAd` já sabe consumir via `location.state` (linha 1272-1289).

**Resultado esperado:** 1 clique no sino = CreateAd aberto no passo 1 com tudo pré-preenchido. A Cris só revisa e publica.

---

### 2. Dashboard mostra data fixa "14 de abril" hardcoded
**Onde:** `frontend/src/pages/Dashboard.jsx:739`
**O que acontece:** Botão do cabeçalho diz **"Hoje, 14 de abril"** e a saudação `isToday = day === 14` (linha 353 no MiniCalendar) também está fixada em 14. Hoje é 18/04/2026.

**Por que dói na Cris:** Primeira impressão do sistema é uma data errada. Ela pensa "esse painel não é do dia de hoje?". Mina confiança nos dados que vêm depois. AdManager sem data correta = cheiro de mock.

**Conserto:**
```jsx
const hoje = new Date();
const diasBr = ['domingo','segunda','terça','quarta','quinta','sexta','sábado'];
// botão: `Hoje, ${hoje.getDate()} de ${mesBr[hoje.getMonth()]}`
// isToday: key === todayStr (já existe no Calendar.jsx:266 — reusar)
```

## 🟠 Alto

### 3. Botões do cabeçalho do Dashboard não fazem nada
**Onde:** `Dashboard.jsx:732-751`
**O que acontece:** "Hoje, 14 de abril" e "Comparar período" são botões (tem borda, cursor pointer, chevron) mas **sem `onClick`**. Clique → silêncio.

**Por que dói:** Affordance quebrada. Ensina a Cris que "botão aqui não é de confiar" — ela passa a hesitar em TODOS os outros botões. Confiança é frágil.

**Conserto:** ou implementar (seletor de data, comparativo período) ou remover. Não deixar botão morto.

---

### 4. MiniCalendar do Dashboard tem cursor de clique mas não clica
**Onde:** `Dashboard.jsx:362`
**O que acontece:** `cursor: event ? 'pointer' : 'default'` — o cursor muda quando há evento, mas a célula não tem `onClick`. Outro clique morto.

**Conserto:** adicionar `onClick` que abra o mesmo modal de data comercial do Calendar.jsx, ou remover o `cursor: pointer`.

---

### 5. Dashboard não mostra datas comerciais — só mostra anúncios agendados
**Onde:** `Dashboard.jsx:71-75` (CALENDAR_EVENTS hardcoded) vs `Calendar.jsx:241` (getUpcomingCommercialDates)
**O que acontece:** MiniCalendar do Dashboard mostra eventos fake ("Esmaltes Tendência", "Skincare Rotina Diária"). Mas as **datas comerciais do Brasil** (Dia das Mães, Black Friday etc — o ativo mais valioso do sistema) só aparecem em `/calendario`. A Cris que passa 30 segundos no Dashboard nunca as vê.

**Por que dói:** O Dashboard é "o que pega atenção em 3s". Hoje ele pega atenção com dados mock. O sinal de valor real (oportunidades pagas próximas) fica escondido a 1 clique de distância.

**Conserto:** Substituir (ou complementar) o MiniCalendar pela lista das 3 próximas datas comerciais — mesmo componente do painel lateral do Calendar.jsx:516-589.

---

### 6. Calendar quebra em mobile
**Onde:** `Calendar.jsx:284`
```jsx
gridTemplateColumns: '1fr 300px'
```
**O que acontece:** Não há media query. Em 375px o painel de 300px (próximos anúncios + datas comerciais) não cabe lado a lado com o calendário. Vai dar overflow horizontal ou o painel direito some.

**Por que dói:** A Cris abre o calendário no celular pra ver "o que tem nos próximos 45 dias". Se o painel direito quebrar, ela perde exatamente a função mais útil da tela.

**Conserto:**
```jsx
@media (max-width: 900px) {
  grid-template-columns: 1fr;
}
```
🔵 **Precisa validação visual** — Claude Chrome pode confirmar o sintoma em 375px.

## 🟡 Médio

### 7. Saudação "Bom dia" hardcoded
**Onde:** `Dashboard.jsx:725`. Se a Cris abre 21h, vê "Bom dia". Trivial consertar com `new Date().getHours()`.

### 8. CreateAd — mesmo com pré-preenchimento, obriga 5 passos
**Onde:** `CreateAd.jsx:28` — `STEPS = ['Objetivo','Público','Orçamento','Criativo','Revisar']`
**Observação:** Quando vem de data comercial, objetivo + orçamento + data + texto já estão preenchidos. Ainda assim a Cris clica Próximo × 4 até chegar em Revisar.
**Sugestão:** se `location.state.commercialDate` presente, abrir direto no passo "Revisar" com botão secundário "Personalizar" que volta para Objetivo. Reduz de ~5 cliques para 1.

### 9. Placeholder da busca tem jargão
**Onde:** `App.jsx:241` — `"Buscar sessões, ajuda ou perguntar à IA..."`. "Sessões" é termo de dev. Trocar por "páginas" ou "telas". A Cris pensa em "telas", não sessões.

### 10. Inconsistência de terminologia: "Novo agendamento" vs "Criar anúncio"
**Onde:** `Calendar.jsx:438` usa "Novo agendamento"; o resto do sistema usa "Criar anúncio". Dois nomes para a mesma ação = carga cognitiva.

### 11. MetricCard ignora o campo `sub`
**Onde:** `Dashboard.jsx:403-441` — o objeto `MOCK_METRICS` passa `sub: 'vs. ontem'` mas o componente não renderiza. O delta aparece mas o contexto "vs ontem" some. Sem esse contexto, "+12,5%" fica ambíguo (vs semana? mês? ontem?).

## 🟢 O que está bom — preservar

- **Modal de data comercial** (`Calendar.jsx:18-173`) — excelente densidade: por que importa, ações sugeridas, orçamento em R$, diretrizes de comunicação com lista "Evite: promoção/desconto". Esse é o diferencial do sistema.
- **Sino animado com badge e shake** (`App.jsx:375`) — affordance correta, badge "9+" bem tratado.
- **Empty state do sino** (`App.jsx:122-125`) — "🔕 Nenhuma notificação no momento" simples e humano.
- **BalanceCard com estado vermelho quando baixo** — feedback visual forte, texto claro "Abaixo do mínimo de R$ 20,00".
- **Busca com fallback para IA** — quando não acha rota, manda pra IA. Redução de frustração de "não achei".
